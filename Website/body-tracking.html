Body Tracking

Body tracking is a program that can identify a body and track its movements, by identifying keypoints in the body and mapping together a skeletal image. This will be used in this project in order to see where each person in the lab is moving in relation to the positions of the room.

There are multiple libraries available for this and I will examine each of them below (descriptons taken from official websites with links suplpied):

OpenPose

OpenPose represents the first real-time multi-person system to jointly detect human body, hand, facial, and foot keypoints (in total 135 keypoints) on single images.
  • Maps a large amount of keypoints to add additional accuracy
  • Works in real time
  • Can identify more than one person 
  • Quite intensive on the GPU
  • Requires a large dataset, but its pre-made
  • Only available on certain devices and languages
  • C++ (main language)
  • Python (API)
  • Command line
  • Any OS
Tensorflow PoseNet PoseNet is a vision model that can be used to estimate the pose of a person in an image or video by estimating where key body joints are.
  • Maps keypoints to essential joints
  • Available on various languages/devices 
  • Have to program and train the program to use the model
  • Less keypoints, thus less accuracy
  • Not reliabel for accuracy 
  • Any language/device as long as you can use the training set

After examining the benefits and drawbacks of each library available, I have decided to use OpenPose for the body tracking library. This is because it is well-known to be a reliable library that has an accurate dataset, and doesn't require much programming to implement.
