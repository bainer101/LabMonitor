Created by Alexander Baine

This project allows users to monitor their labs by providing a live video feed, a feature to download frames for your own processing and providing facial encodings of people in the room for your own processing. This project has been developed so that the users can use the data it provides to suit their unique needs.

When I approached this project, the first thing I had to do was research. I had to break down each part of the project into seperate portions, such as facial recognition and streaming video. I researched and implemented what I learnt separately. This allowed me to see if I could get the basic functionality in the project, and I had the adequate knowledge, before I tried to connect everything together to form a large project.

If the code I implemented for these individual scripts failed, then I would continue my research into the project, but instead of following tutorials as I had been doing previously, I would be looking for StackOverflow questions that are similar to my problems, which would almost always fix the issue.

Once I got all of these individual scripts working, I would combine them together one by one. The Flask server was the most important script as this is how everything was connected, so I would try to connect everything to the Flask website, which came with issues because connecting these scripts to the Flask server came with a lot more topics and issues that I had to research.

The Flask server required a video stream in it, so that was the first to be implemented, which was pretty straight forward as I followed a useful tutorial on it. After this I added the feature to download a frame from the stream, which I had researched prior to implementation. This was implemented after the video stream because it requires the video stream to get the frames.

Once downloading frames was implemented, I was able to start working on facial encodings. Since this was on another device, I needed a way to retrieve the frames from the Flask server, which was through the download frames feature. This means that this part of the project required the ability to download frames to function correctly.

The best advice I can give for working on a project like this is to separate each part of your project into managable topics that you then research and implement separate from each other, then once they're working on their own, make them work together. It is extremely important that you know what order everything needs to be implemented in, like I did here.

All results that I got from this project were what I required, except for body tracking. I wasn't able to install OpenPose, the module required for this, successfully. I installed it a while ago but never tested/verified the installation, which lead to problems when I actually started developing that part of the project. This was a large mistake I made, and I didn't give enough time for myself to fix issues like this if they arose. I would advise, if you are working on something similar, test everything, even if you don't need it for a while, because something like this could happen and can lead to issues.
